Tic-Tac-Toe Game

A Tic Tac Toe game created using HTML, CSS, and JavaScript is a web-based interactive version of the classic game, designed for two players who take turns marking the spaces in a 3x3 grid. The goal is to place three of their marks in a horizontal, vertical, or diagonal row to win the game.

  • HTML: Used to structure the basic layout of the game, including the 3x3 grid and buttons for starting/restarting the game.
  • CSS: Styles the appearance of the game, including the grid, cells, and text. CSS is also used to make the game responsive to different screen sizes and enhance the user interface with colors, fonts, and transitions.
  • JavaScript: Provides the interactivity of the game. It handles the game logic, such as tracking players' turns, checking for a win or a tie, and updating the HTML elements with the players' marks. JavaScript also controls the game's state, enabling dynamic resets and updates without needing to reload the page.

Color Game

A color game created using Python is a simple and interactive desktop or web-based application designed to test and enhance the player's ability to quickly recognize and respond to different colors. This game can be built using Python libraries such as Tkinter for a desktop version

Objective:: Players must select or state the color of the text, not the word written. This challenges the player’s ability to distinguish and react based on the visual cue rather than the word.


The game is typically implemented with:

  • Python Core: Handling the game logic, score keeping, and progression of levels.
  • Tkinter: Creating the graphical user interface (GUI), including buttons, labels for displaying color names, and dialog boxes for interaction.

    • Netflix clone

    A Netflix clone created using HTML, CSS, JavaScript, and APIs is a web-based application designed to mimic the core features of the popular streaming service, Netflix. This project is an excellent way to learn and demonstrate skills in frontend development and API integration.

  • HTML: Used to structure the basic layout of the application, including navigation bars, a main content area for video display, and sections for different categories of videos like "New Releases," "Trending Now," and "Watch Again."
  • CSS: Styles the application to look visually appealing and similar to Netflix's sleek, modern design. CSS is used to manage layouts, colors, fonts, and responsive design features that ensure the site works on devices of all sizes.
  • JavaScript: Adds interactivity to the clone. JavaScript is used for handling user inputs, such as clicking on a movie thumbnail to play a trailer or navigating through different categories. JavaScript also dynamically updates the HTML content based on user interaction and data received from the server.
  • API Fetching in JavaScript: The clone uses JavaScript to fetch data from movie database APIs like The Movie Database (TMDB) API. This is crucial for populating the clone with real movie and TV show data, including titles, images, descriptions, and possibly even video trailers. API calls are made using AJAX, Fetch API, or other methods to retrieve JSON data and then dynamically update the webpage without reloading.

    • Stone-paper-Scissor Game

    A Stone, Paper, Scissors game created using HTML, CSS, and JavaScript is an interactive web-based version of the classic hand game where players select between stone (rock), paper, or scissors. This game is designed for a single player against the computer.


  • HTML: Structures the basic layout of the game, including elements such as buttons for each choice (stone, paper, scissors), a section to display the player's choice, the computer's choice, and the result of the game.
  • CSS: Styles the game to make it visually appealing and engaging. CSS is used to design buttons, add animations, and format text and layouts so that the game is responsive and works well on different devices.
  • JavaScript: Adds the interactivity needed for the game mechanics. JavaScript handles user inputs (the player’s choice) and generates the computer's random choice. It also contains the logic to determine the winner based on the rules (rock crushes scissors, scissors cuts paper, and paper covers rock). After each round, JavaScript updates the webpage to show the choices made, the outcome of the round, and updates the score.

    • The game typically starts with a welcome screen, followed by the main interactive area where the player selects their option. The computer's choice is generated randomly. The results are then compared, and the webpage is updated to display who won that round along with the current score.

    Temperature conversion

    A temperature converter created using HTML, CSS, and JavaScript is a simple web-based application designed to convert temperatures between different scales, commonly Celsius, Fahrenheit, and Kelvin. This utility helps users quickly perform temperature conversions, useful in many educational, scientific, and cooking contexts.

  • HTML: Structures the basic layout of the application. This includes input fields for the user to enter a temperature, radio buttons or a dropdown menu to select the input and output temperature scales, and a button to perform the conversion.
  • CSS: Styles the application to make it visually appealing and user-friendly. CSS is used to design the layout, spacing, colors, and fonts, making the converter easy to navigate and pleasant to use. Responsive design practices ensure that the converter works well on both desktops and mobile devices.
  • JavaScript: Adds interactivity to the application. JavaScript is responsible for handling user inputs, performing the conversion calculations based on the selected scales, and displaying the conversion results. The logic involves:
  • Reading the input temperature and the selected scales from the user input.
  • Calculating the converted temperature using the appropriate formulas (e.g., Celsius to Fahrenheit, Fahrenheit to Kelvin, etc.).
  • Outputting the result dynamically on the webpage without reloading.

    • Toggle-Eye

    A "Toggle Eye" project created using HTML, CSS, and JavaScript is designed to allow users to switch between dark mode and light mode on a webpage. This feature enhances user experience by offering a visually comfortable browsing environment in various lighting conditions. Dark mode reduces the light emitted by device screens while maintaining the minimum color contrast ratios required for readability, which is preferable in low-light conditions. Conversely, light mode is better suited for bright environments.

  • HTML: Structures the basic elements of the webpage, including the toggle switch that users interact with to change the theme. This might be implemented as a simple button, a checkbox, or a custom-designed toggle switch.
  • CSS: Defines the styles for both the light mode and the dark mode. This includes background colors, text colors, button styles, and other elements that change appearance based on the selected mode. CSS variables can be particularly useful here, as they allow you to easily switch values for various properties based on the current theme.
  • JavaScript: Adds the functionality to the toggle switch. When the toggle is activated (e.g., clicked by the user), JavaScript dynamically changes the styles of the webpage elements by altering the CSS classes applied to the body of the document or by switching the values of CSS variables. This change can be animated for a smoother transition effect.
